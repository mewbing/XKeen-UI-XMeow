**Gathered:** 2026-03-01

<domain>
## Phase Boundary

Полноценный YAML-редактор на Monaco Editor с табами для конфигурационных файлов mihomo (config.yaml, ip_exclude, port_exclude, port_proxying). Валидация YAML, кнопки Apply/Save/Format и live-лог панель для отслеживания реакции mihomo после применения конфига. Визуальный редактор правил (Phase 6), редактор групп (Phase 7) — отдельные фазы.

</domain>

<decisions>
## Implementation Decisions

### Раскладка редактора
- Редактор сверху, лог-панель снизу (как в VS Code)
- Между ними — ресайз-сплиттер, пользователь тянет границу
- Лог-панель можно свернуть/развернуть кнопкой — при свёрнутом логе редактор занимает всё пространство
- Тулбар над редактором: табы слева, кнопки (Apply / Save / Format) справа

### Поведение табов
- 4 таба: config.yaml, ip_exclude, port_exclude, port_proxying
- При переключении таба с несохранёнными изменениями — диалог подтверждения «Сохранить / Отменить / Продолжить»
- Индикация несохранённых изменений — цветная точка на табе (как в VS Code)
- Monaco автоопределяет формат файла (YAML для config, plain text для списков)
- Клавиатурные сокращения: Ctrl+1..4 переключает табы, Ctrl+S сохраняет

### Валидация и ошибки
- Инлайн-подсветка ошибок в строке + маркеры на полях (как в VS Code)
- Валидация запускается при Save/Apply (не live при вводе)
- Ошибка валидации НЕ блокирует Apply — показывает предупреждение, но разрешает применить
- YAML-синтаксис + базовая проверка обязательных полей mihomo (mixed-port, dns) и типов значений

### Workflow сохранения
- **Save** (Сохранить) — сохраняет файл на диск без перезапуска mihomo
- **Apply** (Применить) — сохраняет файл на диск И перезапускает mihomo
- Перед Apply — модальное окно diff (inline) с кнопками «Применить» и «Отмена»
- **Format** — форматирует (prettify) содержимое YAML
- Diff включён по умолчанию, можно отключить в настройках; при включённом показывает подсказку «Можно отключить в настройках»

### Diff-превью
- Inline diff (удалённые красным, добавленные зелёным) — не side-by-side
- Отображается в модальном окне поверх редактора
- Кнопки «Применить» / «Отмена» в модальном окне — diff работает как шаг подтверждения
- Включён по умолчанию, отключается в настройках

### Live-лог панель
- Показывает логи только после нажатия Apply (не постоянный стрим)
- Стримит логи mihomo пока пользователь не нажмёт «Стоп» или не свернёт панель
- Каждая строка: цветной бейдж уровня [INFO] [WARN] [ERROR] + текст сообщения
- Авто-скролл всегда к последнему логу (без «умного» паузы при скролле вверх)
- Результат Apply (успех/ошибка перезапуска) показывается как первая запись в лог-панели
- Фильтры: по уровню (info/warning/error) + поле поиска по тексту лога
- Логи очищаются автоматически при новом Apply

### Клавиатурные сокращения
- VS Code-стиль: Ctrl+S сохранить, Ctrl+Z/Y undo/redo, Ctrl+F поиск, Ctrl+Shift+F форматирование, Ctrl+1..4 табы
- Ctrl+Enter — Apply (применить конфиг)
- Подсказки хоткеев на кнопках не нужны

### Claude's Discretion
- Конкретная библиотека для ресайз-сплиттера
- Дизайн диалогов подтверждения
- Размер лог-панели по умолчанию
- Стиль и цвета бейджей уровней логов
- Реализация авто-скролла в лог-панели
- Перехват Ctrl+S браузерного сохранения

</decisions>
