**A. Создать `src/components/config-editor/EditorLogPanel.tsx`:**

Лог-панель с on-demand WebSocket стримингом (НЕ useMihomoWs -- тот подключается автоматически при mount, а нам нужно ручное управление).

1. **WebSocket подключение:**
   - Хук `useEditorLogStream()` внутри компонента (или рядом как отдельный хук):
     - `wsRef = useRef<WebSocket | null>(null)`
     - `start()`: очистить logEntries в store (startLogStream()), построить WS URL из useSettingsStore (mihomoApiUrl, mihomoSecret): `ws(s)://host/logs?token=secret&level=debug`. Открыть WebSocket. Ждать onopen. ws.onmessage: JSON.parse event.data, вызвать addLogEntry({ time, level: data.type, message: data.payload }).
     - `stop()`: ws.close(), wsRef.current = null, stopLogStream() в store
     - ВАЖНО (Pitfall 5 из RESEARCH.md): WS подключается ДО вызова Save+Restart. Поэтому start() вызывается снаружи перед Apply workflow.
   - Cleanup на unmount: stop()

2. **Отображение логов:**
   - Читать logEntries из store (useConfigEditorStore)
   - Каждая запись: `[time] [level] message` в одну строку
   - Level badge: цветной текст -- info: синий, warning: жёлтый, error: красный (простые Tailwind классы, без отдельного компонента)
   - Авто-скролл: useRef на контейнер, useEffect при изменении entries.length -> scrollTo bottom если был внизу. Паттерн из LogStream.tsx (isAtBottomRef).
   - Для небольшого количества записей (до 500) не нужна виртуализация -- простой overflow-auto div

3. **Тулбар лог-панели (встроен в заголовок):**
   - Слева: заголовок "Логи" + количество записей
   - По центру/справа: кнопки-бейджи уровней для фильтрации (info/warning/error) -- клик toggle уровень, по паттерну из LogsToolbar.tsx
   - Справа: кнопка "Стоп" (Square icon) при streaming=true. При нажатии -> stop()
   - Справа: кнопка свернуть/развернуть (ChevronDown/ChevronUp icon) -- вызывает коллбэк onToggleCollapse (от родителя, управляет ResizablePanel)

4. **Фильтрация по уровню:**
   - Локальный state `activeLevels: Set<string>` (по умолчанию все включены)
   - Фильтровать logEntries перед рендером

5. **Props:**
   - `onToggleCollapse: () => void` -- сворачивание панели
   - `collapsed: boolean` -- текущее состояние

6. **Expose:** start() и stop() через `forwardRef + useImperativeHandle` или через props/callbacks. Лучший вариант: передать `startStream` и `stopStream` callbacks наверх через useCallback, которые ConfigEditorPage вызовет в Apply workflow.

Альтернативный дизайн: вместо imperative handle, сделать logPanel stateless -- start/stop управляется через store (logStreaming boolean). useEffect внутри EditorLogPanel: если logStreaming стал true -> connect WS, если false -> disconnect. Это чище и в духе проекта (Zustand-driven).

Использовать Zustand-driven подход: EditorLogPanel подписывается на `logStreaming` из store. Когда true -- подключить WS. Когда false -- отключить. ConfigEditorPage вызывает `startLogStream()` в store перед Apply -> EditorLogPanel реагирует.

4. **Apply workflow (ключевая логика):**
   Когда пользователь нажимает Apply (через EditorToolbar -> ApplyConfirmDialog):

   a. Проверить настройку showDiffBeforeApply из settings store:
      - Если включена: показать DiffPreview. При подтверждении в DiffPreview -- продолжить.
      - Если выключена: показать ApplyConfirmDialog напрямую.

   b. После подтверждения (из DiffPreview или ApplyConfirmDialog):
      1. `startLogStream()` -- переключить logStreaming в store -> EditorLogPanel подключит WS
      2. Если лог-панель свёрнута -- развернуть её (panel.expand())
      3. Подождать ~500ms чтобы WS успел подключиться (или лучше: startLogStream, а save делать с небольшой задержкой)
      4. Сохранить файл: `saveConfig(content)` / `saveXkeenFile(tabId, content)`
      5. `markSaved(activeTab)` в store
      6. Перезапустить mihomo: `serviceAction('restart')` из config-api.ts
      7. toast.success("Конфиг применён, mihomo перезапускается")
      8. Логи будут приходить через WS в лог-панель

   При ошибке на любом шаге -- toast.error, stopLogStream()

5. **Save workflow:**
   Когда EditorToolbar вызывает Save:
   - saveConfig / saveXkeenFile
   - markSaved(activeTab)
   - toast.success

6. **Navigation guard:**
   - `useBeforeUnload` из React Router (или `window.onbeforeunload`): если hasDirtyTabs() -- показать браузерное предупреждение
   - useEffect: `window.onbeforeunload = (e) => { if (store.hasDirtyTabs()) { e.preventDefault() } }`. Cleanup на unmount: `window.onbeforeunload = null`

7. **Collapse state для лог-панели:**
   - useState `logPanelCollapsed: boolean` (default: false)
   - При collapse лог-панели: если streaming -- stopLogStream()
   - Передать collapsed и onToggleCollapse в EditorLogPanel
   - Использовать onCollapse/onExpand callbacks от ResizablePanel
