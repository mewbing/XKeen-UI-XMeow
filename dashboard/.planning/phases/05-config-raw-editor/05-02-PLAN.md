  <name>Task 1: Monaco Editor обёртка с валидацией и keyboard shortcuts</name>
  <files>src/components/config-editor/ConfigEditor.tsx</files>
  <action>
Создать `src/components/config-editor/ConfigEditor.tsx` -- обёртка Monaco Editor, которая:

1. **Импорт Monaco Editor:**
   ```typescript
   import Editor from '@monaco-editor/react'
   import type { editor, Monaco } from 'monaco-editor'
   ```

2. **Refs для инстанса:** `editorRef = useRef<editor.IStandaloneCodeEditor>`, `monacoRef = useRef<Monaco>`.

3. **Подключение к store:** Из `useConfigEditorStore` брать `activeTab`, `tabs[activeTab].current`, `tabs[activeTab].language`, `setContent`, `setValidation`.

4. **Monaco Editor props:**
   - `height="100%"` (родитель должен иметь flex-1 min-h-0)
   - `language` из tabs[activeTab].language ('yaml' для config, 'plaintext' для остальных)
   - `path` = `file:///${activeTab}` -- уникальный path для каждого таба, чтобы Monaco не путал модели между табами (критичный pitfall из RESEARCH.md)
   - `value` = tabs[activeTab].current
   - `onChange` = (value) => setContent(activeTab, value ?? '')
   - `theme="vs-dark"`
   - `options`: minimap: false, fontSize: 13, lineNumbers: 'on', wordWrap: 'on', scrollBeyondLastLine: false, automaticLayout: true, tabSize: 2
   - `loading` = `<Skeleton className="h-full w-full" />` (из @/components/ui/skeleton)

5. **onMount callback:**
   - Сохранить editor и monaco в refs
   - Зарегистрировать Ctrl+S shortcut через `editor.addCommand(monaco.KeyMod.CtrlCmd | monaco.KeyCode.KeyS, handleSave)`. handleSave -- проп от родителя (callback).

6. **YAML валидация (debounced):**
   - При каждом onChange, если текущий язык 'yaml': запускать валидацию через setTimeout с debounce 300ms
   - Валидация: `import yaml from 'js-yaml'`, try `yaml.load(content)` -> valid: true + очистить маркеры. catch YAMLException -> valid: false + установить маркер через `monaco.editor.setModelMarkers(model, 'yaml-validation', [marker])` с severity Error, позиция из e.mark (line+1, column+1)
   - Для plaintext -- всегда valid: true, маркеры не устанавливаются
   - Результат валидации -> `setValidation(activeTab, result)` в store

7. **Загрузка содержимого табов:**
   - useEffect при mount: загрузить содержимое активного таба если оно пустое
   - useEffect при смене activeTab: загрузить содержимое если ещё не загружено (original === '' и не loading)
   - Для config таба: `fetchConfig().then(data => setOriginal('config', data.content))`
   - Для остальных: `fetchXkeenFile(tabId).then(data => setOriginal(tabId, data.content))`
   - Во время загрузки: `setLoading(tab, true)`, после: `setLoading(tab, false)`
   - При ошибке загрузки: toast.error с сообщением

8. **Props:**
   - `onSave: () => void` -- callback для Ctrl+S
   - Никаких других пропов -- всё из store

9. **Обернуть в div с `className="flex-1 min-h-0"`** для корректной высоты Monaco (pitfall из RESEARCH.md).

**A. Создать `src/components/config-editor/EditorToolbar.tsx`:**

Тулбар -- горизонтальная полоса над редактором. Слева -- табы, справа -- кнопки и индикатор валидации.

1. **Табы (слева):**
   - 4 кнопки-таба: "config.yaml", "ip_exclude", "port_exclude", "port_proxying"
   - Активный таб: выделенный стиль (bg-muted/bg-background, border-bottom primary)
   - Dirty-индикатор: цветная точка (bg-orange-400, rounded-full, size-1.5) справа от текста таба, видна только когда tab.dirty === true
   - Клик по табу: если текущий таб dirty -- открыть TabSwitchDialog вместо немедленного переключения. Если не dirty -- переключить сразу через setActiveTab().
   - Используем useState для pendingTab (TabId | null) -- таб, на который пользователь хочет переключиться, но текущий dirty

2. **Индикатор валидации (центр/справа от табов):**
   - Для yaml-табов: если valid -- зелёный бейдж "YAML OK". Если invalid -- красный бейдж "Ошибка: строка {line}" с иконкой AlertTriangle
   - Для plaintext-табов: ничего не показывать
   - Данные из store: tabs[activeTab].validation

3. **Кнопки (справа):**
   - **Format** (Wand2 icon): `onClick` -> импортировать js-yaml, вызвать `yaml.dump(yaml.load(content), { indent: 2, lineWidth: 120, noRefs: true, sortKeys: false })`, обновить store через setContent. Неактивна (disabled) для plaintext-табов. При ошибке парсинга -- toast.error("Невозможно отформатировать: невалидный YAML"). ВАЖНО: перед Format показать toast.warning("Форматирование удалит комментарии") -- предупреждение per RESEARCH.md pitfall 4.
   - **Save** (Save icon): Сохраняет текущий файл на сервер. Для config -> saveConfig(content), для остальных -> saveXkeenFile(tabId, content). После успеха -> markSaved(activeTab), toast.success("Сохранено"). Если валидация YAML невалидна -- Save работает, но сначала показать toast.warning("YAML содержит ошибки") перед сохранением.
   - **Apply** (Play icon): Открывает ApplyConfirmDialog. Кнопка должна быть primary/accent стиля.
   - Все кнопки размера sm с текстом и иконками
   - Показать Loader2 spinner вместо иконки когда действие выполняется (loading state)

4. **Keyboard shortcuts** -- useEffect с window keydown listener:
   - `Ctrl+1..4` -> переключить таб (с проверкой dirty как при клике)
   - Не перехватывать Ctrl+S здесь -- это делает Monaco Editor в ConfigEditor.tsx

5. **Props:**
   - `onApplyConfirmed: () => Promise<void>` -- callback когда Apply подтверждён (вызывается из ApplyConfirmDialog)
