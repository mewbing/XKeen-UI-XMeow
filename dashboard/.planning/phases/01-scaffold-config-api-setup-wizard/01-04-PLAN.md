    1. Обновить src/App.tsx для интеграции wizard:
       - Wizard gate: if (!isConfigured) return <SetupWizard />
       - Zustand hydration check: использовать onRehydrateStorage или проверку _hasHydrated
         чтобы избежать мелькания wizard при каждой загрузке у настроенного пользователя.
         Пока store не гидрирован из localStorage, показывать пустой экран или loading spinner.
       - Полный flow:
         1. App рендерится
         2. Zustand начинает hydration из localStorage
         3. Пока hydration -- пустой/loading экран
         4. Hydration завершена, isConfigured=false -> SetupWizard
         5. Hydration завершена, isConfigured=true -> BrowserRouter с AppLayout

    2. Для Zustand hydration -- добавить в settings store:
       ```typescript
       // В store добавить:
       _hasHydrated: false,
       setHasHydrated: (state: boolean) => set({ _hasHydrated: state }),
       // В persist options:
       onRehydrateStorage: () => (state) => {
         state?.setHasHydrated(true)
       }
       ```

    3. В App.tsx:
       ```typescript
       const isConfigured = useSettingsStore((s) => s.isConfigured)
       const hasHydrated = useSettingsStore((s) => s._hasHydrated)

       if (!hasHydrated) {
         return null // или loading spinner
       }

       if (!isConfigured) {
         return <SetupWizard />
       }

       return <BrowserRouter>...</BrowserRouter>
       ```

    4. После успешного wizard (StepSuccess -> "Начать работу"):
       - setConfigured() сохраняет type, mihomoUrl, secret, configUrl
       - isConfigured становится true -> App перерисовывает BrowserRouter -> redirect на /overview
