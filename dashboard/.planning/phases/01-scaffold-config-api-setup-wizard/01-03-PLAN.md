  <action>
    Расширить существующий backend/server.py (из Plan 01) полными API endpoints:

    1. **Конфигурация путей через env vars:**
       - MIHOMO_CONFIG_PATH = '/opt/etc/xkeen/config.yaml' (default)
       - XKEEN_DIR = '/opt/etc/xkeen' (default)
       - BACKUP_DIR = '/opt/etc/xkeen/backups' (default)

    2. **GET /api/config:**
       - Читает CONFIG_PATH с encoding='utf-8'
       - Возвращает {"content": "..yaml text.."} с кодом 200
       - Если файл не найден -- {"error": "Config file not found"}, код 404
       - Если ошибка чтения -- {"error": "..."}, код 500

    3. **PUT /api/config:**
       - Принимает JSON {"content": "..yaml text.."}
       - Шаг 1: Валидация YAML через ruamel.yaml.load(StringIO(content)) в try/except
         - Если невалидный YAML: {"error": "Invalid YAML: {details}"}, код 400
       - Шаг 2: Создание backup (если файл существует):
         - os.makedirs(BACKUP_DIR, exist_ok=True)
         - Имя backup: config_{YYYYMMDD_HHMMSS}.yaml
         - shutil.copy2(CONFIG_PATH, backup_path) -- сохраняет метаданные
       - Шаг 3: Запись нового конфига с encoding='utf-8'
       - Возвращает {"message": "Config saved", "backup": backup_path}, код 200

    4. **GET /api/xkeen/<filename>:**
       - Допустимые filename: ip_exclude, port_exclude, port_proxying
       - Маппинг на файлы: ip_exclude -> ip_exclude.lst, и т.д.
       - Недопустимый filename: {"error": "Unknown file"}, код 404
       - Файл не найден: {"content": "", "filename": "..."}, код 200 (пустое содержимое, не ошибка)
       - Читает с encoding='utf-8'
       - Возвращает {"content": "..text..", "filename": "ip_exclude.lst"}, код 200

    5. **PUT /api/xkeen/<filename>:**
       - Те же допустимые filename
       - Принимает JSON {"content": "..text.."}
       - Создаёт backup: {filename}_{YYYYMMDD_HHMMSS}.lst в BACKUP_DIR
       - Записывает файл с encoding='utf-8'
       - Возвращает {"message": "{filename} saved"}, код 200

    6. **Общая обработка ошибок:**
       - Все endpoint'ы обёрнуты в try/except
       - Ошибки возвращают {"error": "..."} с соответствующим HTTP-кодом
       - Логирование ошибок через app.logger

    ВАЖНО:
    - ruamel.yaml YAML() инициализируется ОДИН раз на уровне модуля с preserve_quotes=True
    - Все open() вызовы с encoding='utf-8' (критично для русских комментариев и emoji в конфиге)
    - НЕ использовать PyYAML (import yaml) -- только ruamel.yaml
    - Backup директория создаётся автоматически при первой записи
