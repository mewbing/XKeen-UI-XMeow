  <action>
    1. Инициализировать проект Vite в текущей директории (dashboard/):
       ```bash
       pnpm create vite@latest . -- --template react-ts
       ```
       ВАЖНО: Если директория не пуста, vite может спросить -- использовать флаг для overwrite или предварительно создать в temp и скопировать.

    2. Установить зависимости:
       ```bash
       pnpm add tailwindcss @tailwindcss/vite
       pnpm add -D @types/node
       pnpm add react-router zustand lucide-react
       ```

    3. Инициализировать shadcn/ui:
       ```bash
       pnpm dlx shadcn@latest init
       ```
       При вопросах: стиль -- "new-york", цветовая схема -- "zinc", CSS variables -- yes.
       Это автоматически создаст components.json, обновит index.css с CSS-переменными темы, создаст src/lib/utils.ts с cn().

    4. Обновить vite.config.ts:
       - Добавить @tailwindcss/vite плагин
       - Добавить path alias: "@" -> "./src"
       - Добавить dev proxy: "/api" -> "http://localhost:5000"
       Использовать точный паттерн из RESEARCH.md (Pattern: Vite Config).
       НЕ создавать tailwind.config.js -- Tailwind v4 использует CSS-first конфигурацию.

    5. Обновить src/App.tsx -- пока простой placeholder:
       ```tsx
       function App() {
         return <div className="min-h-screen bg-background text-foreground">Mihomo Dashboard</div>
       }
       export default App
       ```

    6. Убедиться что src/index.css содержит:
       - `@import "tailwindcss";`
       - `@import "tw-animate-css";` (shadcn/ui init должен добавить)
       - CSS-переменные темы shadcn/ui
       НЕ использовать tailwindcss-animate (deprecated для v4).
  </action>
  <verify>
    ```bash
    pnpm run dev
    ```
    Открыть http://localhost:5173 -- страница отображает "Mihomo Dashboard" с применёнными стилями Tailwind (bg-background, text-foreground).
    Проверить отсутствие ошибок в консоли.
    Проверить что файл tailwind.config.js НЕ создан.
  </verify>
