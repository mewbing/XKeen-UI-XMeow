**4. Create src/stores/update.ts**

Zustand store (NOT persisted -- volatile state). Interface:
```typescript
interface UpdateState {
  releaseInfo: ReleaseInfo | null
  hasUpdate: boolean
  isExternalUI: boolean
  checking: boolean
  applying: boolean
  applyingDist: boolean
  error: string | null

  checkForUpdate: () => Promise<void>
  applyUpdate: () => Promise<void>
  applyDist: () => Promise<void>
  rollback: () => Promise<void>
  clearError: () => void
}
```

Actions:
- `checkForUpdate`: set checking=true, call checkUpdate(), set releaseInfo/hasUpdate/isExternalUI from response, catch -> set error. Always set checking=false in finally.
- `applyUpdate`: set applying=true, call applyUpdate(). On success, do NOT clear applying (overlay will handle health poll). On error, set applying=false, set error.
- `applyDist`: set applyingDist=true, call applyDist(). On success, set applyingDist=false. On error, set applyingDist=false, set error.
- `rollback`: call rollbackUpdate(). On error set error.
- `clearError`: set error=null

Export as `export const useUpdateStore = create<UpdateState>()(...)`.

**2. Initialize auto-check on app load (App.tsx)**

Import `useUpdateStore` and `useSettingsStore` (settings already imported).

Add `useEffect` in App component (after the existing reduceMotion useEffect), INSIDE the `isConfigured` branch (below the HashRouter return, not the wizard return). Instead, add the auto-check logic at the TOP of the App component (before returns) so it runs when configured:

```typescript
const autoCheckUpdates = useSettingsStore((s) => s.autoCheckUpdates)

useEffect(() => {
  if (!isConfigured || !autoCheckUpdates) return

  // Check on load
  useUpdateStore.getState().checkForUpdate()

  // Periodic check every 6 hours
  const CHECK_INTERVAL_MS = 6 * 60 * 60 * 1000
  const id = setInterval(() => {
    if (useSettingsStore.getState().autoCheckUpdates) {
      useUpdateStore.getState().checkForUpdate()
    }
  }, CHECK_INTERVAL_MS)

  return () => clearInterval(id)
}, [isConfigured, autoCheckUpdates])
```

Key: the dependency array includes `autoCheckUpdates` so toggling the setting on/off restarts the interval. And `isConfigured` ensures no checks before setup is complete.
