**3. UpdateOverlay.tsx**

Full-screen overlay component for showing update progress and health polling.

Props:
```typescript
interface UpdateOverlayProps {
  open: boolean
  mode: 'server' | 'dist'    // server = binary update (needs restart), dist = SPA files only
  onClose: () => void
}
```

State:
- `phase`: 'downloading' | 'installing' | 'restarting' | 'done' | 'error' | 'timeout'
- `error`: string | null
- `logs`: string[] -- log messages

Behavior:
- When `open` becomes true, start the update flow:
  - If mode='server': call `useUpdateStore.getState().applyUpdate()`.
    - Show phases: "Скачивание обновления..." -> "Установка..." -> "Перезапуск сервера..."
    - After applyUpdate resolves (status: "restarting"):
      - Wait 3 seconds (avoid false-positive from old server still alive)
      - Then poll `checkHealth()` every 2 seconds, max 15 attempts (30s total)
      - If health OK: set phase='done', call `window.location.reload()` after 500ms
      - If timeout: set phase='timeout'
    - If applyUpdate throws: set phase='error', store error message
  - If mode='dist': call `useUpdateStore.getState().applyDist()`.
    - Show: "Обновление дашборда..." -> "Готово!"
    - On success: set phase='done', show "Дашборд обновлён" message, auto `window.location.reload()` after 2 seconds
    - On error: set phase='error'

Render: Fixed overlay covering entire screen (`fixed inset-0 z-50 bg-background/95 backdrop-blur-sm`):
- Center content vertically and horizontally
- Spinner (Loader2 animate-spin) while in downloading/installing/restarting phases
- CheckCircle2 icon (green) in done phase
- XCircle icon (red) in error phase
- Status text in large font below icon
- Log area: scrollable div with monospace text showing log entries
- **Error state**: Show error message + "Закрыть" button calling onClose
- **Timeout state**: Show "Сервер не отвечает" + "Перезагрузить вручную" button that calls `window.location.reload()` + "Закрыть" button
- **Done state**: Show "Перезагрузка..." briefly before reload happens

CRITICAL timing per RESEARCH.md Pitfall 3: After receiving "restarting" response, wait 3 seconds before first health poll to avoid false-positive from old server still alive.

CRITICAL per RESEARCH.md Pitfall 6: After apply-dist success, do NOT poll health (server never went down). Just show success and reload after 2s.
