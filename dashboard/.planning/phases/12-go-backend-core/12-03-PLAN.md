**internal/logwatch/hub.go**:
- `type LogHub struct`:
  - `mu sync.RWMutex`
  - `clients map[*Client]struct{}`
  - `cfg *config.AppConfig`
  - `watcher *Watcher` -- nil when no clients
  - `stopCh chan struct{}` -- signals watcher goroutine to stop
- `func NewLogHub(cfg *config.AppConfig) *LogHub`
- `func (h *LogHub) AddClient(c *Client)`:
  - Lock, add client to map
  - If first client (len == 1): start watcher
- `func (h *LogHub) RemoveClient(c *Client)`:
  - Lock, delete client from map
  - If last client (len == 0): stop watcher
- `func (h *LogHub) BroadcastNewLines(filename string)`:
  - RLock, iterate clients
  - For each client where `client.currentFile` matches filename (minus .log extension):
    - Call `ReadFromOffset` with client's current offset
    - If truncated: send full initial (ReadLogTail) and reset offset
    - If new lines: send `{"type": "append", "lines": [...]}`
    - Update client offset
  - Use goroutine per client send to avoid blocking (or accept sequential for simplicity with small client count)
- `func (h *LogHub) Shutdown()`:
  - Stop watcher if running, close all client connections

**internal/logwatch/watcher.go**:
- `type Watcher struct`:
  - `fsw *fsnotify.Watcher` -- may be nil if fsnotify fails
  - `ticker *time.Ticker` -- polling fallback
  - `stopCh chan struct{}`
  - `hub *LogHub`
  - `logDir string`
  - `usingPolling bool`
- `func NewWatcher(hub *LogHub, logDir string, allowedLogs map[string]string) *Watcher`:
  - Try `fsnotify.NewWatcher()`. If error (e.g., inotify unavailable on Entware): set usingPolling=true, log warning
  - If fsnotify works: for each log file in allowedLogs, check if file exists. If exists, `watcher.Add(filePath)`. If not, `watcher.Add(logDir)` to catch file creation.
  - If polling: create `time.NewTicker(500 * time.Millisecond)` -- per RESEARCH recommendation
- `func (w *Watcher) Start()`:
  - If fsnotify: goroutine watching `w.fsw.Events` channel
    - On `fsnotify.Write` event: `w.hub.BroadcastNewLines(event.Name)` (extract base filename)
    - On `fsnotify.Create` event: if filename matches allowed log, add to watcher
  - If polling: goroutine with ticker, on each tick check file sizes for all allowed logs. If size changed, broadcast.
  - Both modes: select on stopCh for clean exit
- `func (w *Watcher) Stop()`:
  - Signal stopCh
  - If fsnotify: `w.fsw.Close()`
  - If polling: `w.ticker.Stop()`

**internal/handler/ws_logs.go** -- port from server.py lines 549-640:
- `var upgrader = websocket.Upgrader{CheckOrigin: func(r *http.Request) bool { return true }}`
- `type WsLogHandler struct`:
  - `hub *logwatch.LogHub`
  - `cfg *config.AppConfig`
- `func NewWsLogHandler(hub *logwatch.LogHub, cfg *config.AppConfig) *WsLogHandler`
- `func (h *WsLogHandler) ServeHTTP(w http.ResponseWriter, r *http.Request)`:
  1. Upgrade connection: `upgrader.Upgrade(w, r, nil)`. On error: return.
  2. Create `logwatch.Client` with conn, currentFile="error"
  3. `h.hub.AddClient(client)`, defer `h.hub.RemoveClient(client)`
  4. Send initial content:
     - `lines, offset := logwatch.ReadLogTail(h.cfg.XkeenLogDir, "error", h.cfg.AllowedLogs, 1000)` (1000 lines per CONTEXT decision)
     - `client.SendJSON(map[string]interface{}{"type": "initial", "lines": lines, "file": "error"})`
     - Set client.offset = offset
  5. Read loop (blocking):
     - `_, message, err := conn.ReadMessage()` -- blocks until message or close
     - On error: break (client disconnected)
     - Parse JSON: `{"type": "...", "file": "..."}`
     - Switch on type:
       - `"switchFile"`: validate file name against AllowedLogs, strip ".log" suffix. Update client.currentFile. Read tail, send initial, update offset.
       - `"reload"`: read tail for current file, send initial, update offset.
       - `"clear"`: call `logwatch.ClearLog(...)`, send `{"type": "clear"}`, reset offset to 0.
       - `"ping"`: send `{"type": "pong"}`
  6. After loop exits (client disconnected): RemoveClient runs via defer
