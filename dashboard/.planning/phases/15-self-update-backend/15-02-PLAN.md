**internal/handler/update.go:**

Создать отдельный handler struct для update endpoints (отделён от основных Handlers, т.к. имеет зависимость на updater.Updater).

`UpdateHandler` struct:
- updater *updater.Updater

`NewUpdateHandler(upd *updater.Updater, cfg *config.AppConfig) *UpdateHandler`

**Endpoint: GET /api/update/check — CheckUpdate handler:**
- ctx с 15s timeout: `ctx, cancel := context.WithTimeout(r.Context(), 15*time.Second)`
- Вызвать u.updater.Check(ctx)
- При ошибке → writeJSON(500, {"error": err.Error()})
- При успехе → writeJSON(200, releaseInfo) — ReleaseInfo уже имеет JSON теги

**Endpoint: POST /api/update/apply — ApplyUpdate handler:**
- Проверить u.updater.IsUpdating() → если да, writeJSON(409, {"error": "update already in progress"})
- ctx с 5min timeout для долгого скачивания: `context.WithTimeout(r.Context(), 5*time.Minute)`
- Вызвать u.updater.Apply(ctx)
- При ошибке → writeJSON(500, {"error": err.Error()})
- При успехе → writeJSON(200, {"status": "restarting", "message": "Update applied, restarting..."})
- CRITICAL: response.Write ПЕРЕД restart! Использовать `w.(http.Flusher).Flush()` если доступен
- `time.AfterFunc(1*time.Second, func() { restartService() })` — задержка для отправки ответа
- Функция `restartService()`:
  - initdScript := "/opt/etc/init.d/S99xmeow-ui"
  - Проверить os.Stat(initdScript) — если не найден, log.Printf предупреждение (ответ уже отправлен, ошибку вернуть нельзя)
  - `exec.Command(initdScript, "restart").Start()` — fire-and-forget (init.d убьёт текущий процесс)

**Endpoint: POST /api/update/rollback — RollbackUpdate handler:**
- Вызвать u.updater.Rollback()
- При ошибке → writeJSON(500, {"error": err.Error()})
- При успехе → writeJSON(200, {"status": "restarting", "message": "Rollback applied, restarting..."})
- Flush + time.AfterFunc(1*time.Second, restartService) — аналогично apply

**Endpoint: POST /api/update/apply-dist — ApplyDist handler (SUPD-06):**
Скачивает dist.tar.gz и распаковывает в external-ui директорию mihomo.
- Проверить u.updater.IsExternalUI() → если false, writeJSON(400, {"error": "not in external-ui mode"})
- Проверить u.updater.IsUpdating() → 409
- ctx 5min timeout
- Вызвать u.updater.Check(ctx) — получить info с DistURL
- Если DistURL пустой → writeJSON(404, {"error": "dist.tar.gz not found in release"})
- externalUIDir := config.ReadMihomoField(u.cfg.MihomoConfigPath, "external-ui")
- Если пустой → ошибка
- Resolve externalUIDir relative to mihomo config dir: `filepath.Join(filepath.Dir(u.cfg.MihomoConfigPath), externalUIDir)`
- checkDiskSpace для externalUIDir
- downloadFile(ctx, distURL, tmpPath) в externalUIDir
- verifyChecksum (аналогично — скачать checksums.txt, проверить "dist.tar.gz")
- extractDistTarGz(tmpPath, externalUIDir) — перезаписывает содержимое
- cleanup tmp файл
- writeJSON(200, {"status": "ok", "message": "Dashboard updated"})

ПРИМЕЧАНИЕ: функции downloadFile, verifyChecksum, extractDistTarGz, checkDiskSpace из updater пакета. Они не экспортированы по плану 01 (lowercase). Нужно либо:
a) Сделать их экспортированными в Plan 01 (DownloadFile, VerifyChecksum, etc.) — предпочтительный путь
b) Добавить метод ApplyDist() в Updater struct

Решение: Добавить метод `(u *Updater) ApplyDist(ctx context.Context, destDir string) error` в updater.go — он инкапсулирует download+verify+extract dist.tar.gz. Handler вызывает `u.updater.ApplyDist(ctx, externalUIDir)`.

В Plan 01 action уже описано extractDistTarGz — executor должен также экспортировать нужные функции или добавить ApplyDist метод. Добавить в этом task явное указание: если ApplyDist не был создан в Plan 01, создать его в updater.go как часть этого task.
