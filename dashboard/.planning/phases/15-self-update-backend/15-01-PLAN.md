**internal/updater/github.go:**

Создать GitHub API клиент для получения информации о последнем релизе.

Константы:
- `githubRepo = "mewbing/XKeen-UI-Xmeow"` — хардкод, без поддержки форков (locked decision)
- `githubAPI = "https://api.github.com/repos/" + githubRepo + "/releases/latest"`

Типы:
- `githubRelease` struct: TagName, Body, Prerelease bool, PublishedAt time.Time, Assets []githubAsset (json tags: tag_name, body, prerelease, published_at, assets)
- `githubAsset` struct: Name, Size int64, BrowserDownloadURL string (json tags: name, size, browser_download_url)

Функция `fetchLatestRelease(ctx context.Context, userAgent string) (*githubRelease, error)`:
- `http.NewRequestWithContext` на githubAPI
- Headers: `Accept: application/vnd.github.v3+json`, `User-Agent: XMeow-UI/{version}`
- Timeout через ctx (caller передаёт context с 15s timeout)
- Проверить resp.StatusCode == 200, иначе ошибка с кодом (для 403 — rate limit hint)
- Decode JSON в githubRelease
- Пропустить prerelease (release.Prerelease == true → ошибка "no stable release found")

Функция `findAssets(assets []githubAsset) (binaryAsset, checksumAsset, distAsset *githubAsset)`:
- suffix = `runtime.GOOS + "_" + runtime.GOARCH`
- Итерировать assets: если Contains(name, suffix) && HasSuffix(".tar.gz") && name != "dist.tar.gz" → binaryAsset
- Если name == "checksums.txt" → checksumAsset
- Если name == "dist.tar.gz" → distAsset

Функция `compareVersions(current, latest string) int`:
- Strip "v" prefix с обеих строк
- Split by "-" чтобы отделить prerelease суффикс (сравниваем только major.minor.patch)
- `parseVersion(s) [3]int` — split by ".", Atoi каждую часть
- Сравнить числа: -1 (current < latest), 0 (equal), 1 (current > latest)

Функция `downloadFile(ctx context.Context, url, destPath string) error`:
- `http.NewRequestWithContext`, `http.DefaultClient.Do`
- Создать файл через `os.Create(destPath)`
- `io.Copy(file, resp.Body)` — потоковое скачивание
- defer cleanup: если ошибка, `os.Remove(destPath)`
- Проверить resp.StatusCode == 200

**internal/updater/archive.go:**

Утилиты для работы с tar.gz архивами, SHA256 верификации и проверки диска.

Функция `extractBinaryFromTarGz(archivePath, targetName, destDir string) (string, error)`:
- Открыть файл → gzip.NewReader → tar.NewReader
- Итерировать tar entries
- Для каждого entry: `cleanName := filepath.Base(hdr.Name)` — strip directory prefix для безопасности
- Если cleanName == targetName: записать в `destDir + "/." + targetName + ".new"` (скрытый temp файл в том же каталоге, чтобы rename был на одной FS)
- `io.LimitReader(tr, 100*1024*1024)` — максимум 100MB (защита от decompression bomb)
- `os.OpenFile(tmpPath, O_CREATE|O_WRONLY|O_TRUNC, 0755)` — сразу с executable permissions
- Если target не найден в архиве — ошибка
- defer os.Remove на случай ошибки после записи

Функция `verifyChecksum(ctx context.Context, archivePath, checksumURL, archiveName string) error`:
- Скачать checksums.txt во временный файл (в тот же каталог) через downloadFile
- Прочитать содержимое, split по строкам
- Для каждой строки: `strings.Fields(line)` — если len==2 && fields[1]==archiveName → expectedHash = fields[0]
- Если не найден → ошибка "checksum for {name} not found"
- Открыть archivePath, `sha256.New()`, `io.Copy(hasher, file)`
- `hex.EncodeToString(h.Sum(nil))` — сравнить с expectedHash
- Если не совпадает → ошибка "checksum mismatch: expected X, got Y"
- Очистить temp checksums файл

Функция `checkDiskSpace(path string, requiredBytes int64) error`:
- `unix.Statfs(path, &stat)` из `golang.org/x/sys/unix`
- `available := int64(stat.Bavail) * stat.Bsize`
- Минимум: max(requiredBytes, 20*1024*1024) — не менее 20MB
- Если insufficient → ошибка с MB-читаемыми значениями

Функция `extractDistTarGz(archivePath, destDir string) error`:
- Открыть tar.gz
- Итерировать entries: для каждого файла (TypeReg) — создать в destDir
- Для каждой директории (TypeDir) — os.MkdirAll
- Security: `filepath.Clean(filepath.Join(destDir, hdr.Name))` — проверить strings.HasPrefix от destDir (zip slip protection)
- io.LimitReader(tr, 100*1024*1024) для каждого файла

  <name>Task 2: Создать updater/updater.go — Updater struct с кэшем, Check/Apply/Rollback</name>

Создать центральный Updater struct, который собирает всё воедино.

**Типы:**

`ReleaseInfo` struct (экспортируемый, для JSON-сериализации в handlers):
- CurrentVersion string `json:"current_version"`
- LatestVersion string `json:"latest_version"`
- HasUpdate bool `json:"has_update"`
- ReleaseNotes string `json:"release_notes"`
- PublishedAt time.Time `json:"published_at"`
- AssetName string `json:"asset_name"`
- AssetSize int64 `json:"asset_size"`
- DistSize int64 `json:"dist_size,omitempty"` (для Phase 16 UI)
- IsPrerelease bool `json:"is_prerelease"`

`Updater` struct:
- mu sync.RWMutex — для кэша
- cached *ReleaseInfo
- cachedAt time.Time
- cacheTTL time.Duration (1*time.Hour)
- cfg *config.AppConfig
- updating atomic.Bool — блокировка конкурентных обновлений

`NewUpdater(cfg *config.AppConfig) *Updater`:
- Создать с cacheTTL = 1 * time.Hour

**Методы:**

`(u *Updater) Check(ctx context.Context) (*ReleaseInfo, error)`:
1. RLock: проверить cached != nil && time.Since(cachedAt) < cacheTTL → вернуть копию кэша
2. RUnlock, вызвать fetchLatestRelease(ctx, "XMeow-UI/"+u.cfg.Version)
3. Вызвать findAssets для поиска binary/checksum/dist assets
4. Если binaryAsset == nil → ошибка "no binary found for {GOOS}_{GOARCH}"
5. compareVersions(cfg.Version, release.TagName) → HasUpdate = (result < 0)
6. Заполнить ReleaseInfo, Lock → сохранить в кэш → Unlock
7. Вернуть ReleaseInfo

`(u *Updater) Apply(ctx context.Context) error`:
1. `u.updating.CompareAndSwap(false, true)` — если false, return "update already in progress"
2. defer `u.updating.Store(false)`
3. Вызвать Check(ctx) — получить info. Если !HasUpdate → ошибка
4. `os.Executable()` → `filepath.EvalSymlinks()` → exePath
5. destDir := filepath.Dir(exePath)
6. checkDiskSpace(destDir, info.AssetSize * 3) — 3x для архив+бинарник+запас
7. Скачать архив: downloadFile(ctx, assetURL, tmpArchivePath) в destDir (НЕ в /tmp — та же FS для rename)
   - tmpArchivePath = filepath.Join(destDir, ".xmeow-update.tar.gz")
8. verifyChecksum(ctx, tmpArchivePath, checksumURL, assetName)
9. extractBinaryFromTarGz(tmpArchivePath, "xmeow-server", destDir) → newBinaryPath
10. os.Remove(tmpArchivePath) — cleanup архива
11. bakPath := exePath + ".bak"
    - os.Remove(bakPath) — удалить старый бэкап если есть
    - os.Rename(exePath, bakPath) — бэкап текущего
12. os.Rename(newBinaryPath, exePath) — атомарная замена
    - При ошибке: os.Rename(bakPath, exePath) — rollback
13. os.Chmod(exePath, 0755)
14. Инвалидировать кэш: Lock → u.cached = nil → Unlock
15. Return nil

Для external-ui: Check() заполняет DistURL/DistSize из distAsset. Фактическое обновление SPA — Phase 16 через отдельный handler.

`(u *Updater) Rollback() error`:
1. os.Executable() → filepath.EvalSymlinks() → exePath
2. bakPath := exePath + ".bak"
3. Проверить os.Stat(bakPath) — если нет → ошибка "no backup found"
4. os.Remove(exePath) — удалить текущий (может быть повреждён)
5. os.Rename(bakPath, exePath) — восстановить бэкап
6. os.Chmod(exePath, 0755)
7. Return nil

`(u *Updater) IsUpdating() bool` — return u.updating.Load()

`(u *Updater) InvalidateCache()` — Lock → u.cached = nil → Unlock
