**Create `src/components/rules/RuleBlockCard.tsx`:**
A single rule block card (collapsed/expanded).
- Props: `block: RuleBlock`, `index: number`, `density: 'min' | 'detailed'`, `isExpanded: boolean`, `onToggleExpand: () => void`
- **Collapsed state:**
  - Position number badge (`#{index+1}`) left side
  - Block name (bold)
  - Rule count badge (e.g. "12 правил")
  - Target proxy-group badge (colored)
  - If density = 'detailed': show preview of first 2-3 rules (type + value) as small muted text below the name
- **Expanded state:**
  - Same header but with chevron rotated
  - Renders list of RuleRow components for all `block.rules`
  - Use CSS grid-rows animation pattern from ProxyGroupCard (grid-rows-[0fr] → grid-rows-[1fr])
- Card uses `bg-card border rounded-lg` styling consistent with ProxyGroupCard
- The entire card is the drag handle (no separate grip icon — per user decision)
- When dragging (isDragging prop), reduce opacity to 0.5

**Create `src/components/rules/RuleBlockList.tsx`:**
Sortable list wrapping block cards with @dnd-kit.
- Import `DndContext`, `closestCenter`, `DragOverlay` from `@dnd-kit/core`
- Import `SortableContext`, `verticalListSortingStrategy`, `useSortable` from `@dnd-kit/sortable`
- Import `CSS` from `@dnd-kit/utilities`
- Import `restrictToVerticalAxis` from `@dnd-kit/modifiers`
- Create `SortableBlockCard` wrapper component that uses `useSortable({ id: block.id })` and applies transform/transition styles
- `onDragEnd` handler: extract `active.id` and `over.id`, find old/new indices, call `useRulesEditorStore.getState().reorderBlocks(oldIndex, newIndex)`
- DragOverlay renders a clone of the active card for visual feedback
- Props: `blocks: RuleBlock[]`, `density`, `expandedBlocks: Set<string>`, `onToggleExpand: (blockId: string) => void`
- Support 3 layout modes from settings:
  - `list`: single column, vertical scroll (default)
  - `grid`: CSS grid with `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` — expanded card gets `col-span-full`
  - `proxies`: same grid as Proxies page pattern
- Use `restrictToVerticalAxis` modifier for list layout; no modifier for grid layout
- **Inline new block card:** When `rulesNewBlockMode === 'inline'` (from settings store), render a special "new block" placeholder card at the end of the block list. This card has a dashed border (`border-dashed border-muted-foreground/30`), a "+" icon, and text "Новый блок". When clicked, it transforms into an editable card with two inputs: block name (text input) and target proxy-group (Select dropdown from `proxyGroups`). Submit button "Создать" calls `useRulesEditorStore.getState().createBlock(name, target)`, then resets to placeholder state. Escape key or blur cancels editing. When `rulesNewBlockMode === 'dialog'`, this placeholder card is NOT shown (dialog mode uses NewBlockDialog from Plan 03 instead, triggered from toolbar).
