**Step 2:** Create `src/lib/rules-parser.ts` with the following:

**Types:**
```typescript
interface ParsedRule {
  id: string               // unique id for dnd-kit (e.g. "rule-0")
  raw: string              // original string: "DOMAIN-SUFFIX,example.com,ProxyGroup"
  type: string             // "DOMAIN-SUFFIX", "GEOSITE", "AND", "MATCH", etc.
  value: string            // "example.com" (empty for MATCH)
  target: string           // "ProxyGroup"
  noResolve: boolean       // has ",no-resolve" suffix
  commentBefore?: string   // comment lines above this rule (from YAML node)
  commentInline?: string   // inline comment after rule (from YAML node)
}

interface RuleBlock {
  id: string               // unique id for dnd-kit
  name: string             // display name (proxy-group or section name)
  target: string           // primary proxy-group target
  rules: ParsedRule[]      // rules inside this block
  sectionComment?: string  // "# >>> ADULT" style marker text
  startIndex: number       // position in original rules array
}
```

**Functions to implement:**

1. `parseRuleString(raw: string): { type, value, target, noResolve }` — Parse a single rule string. Handle compound rules (AND/OR/NOT with balanced parentheses), MATCH (single arg), and standard TYPE,VALUE,TARGET[,no-resolve]. For compound rules: regex `^(AND|OR|NOT),(\(\(.*\)\)),(.+)$` to extract type, compound body, target. For MATCH: `raw.startsWith('MATCH,')` → type=MATCH, value='', target=rest. For standard: split by comma, last element is target (or second-to-last if last is 'no-resolve').

2. `parseRulesFromConfig(configYaml: string): { doc, rules: ParsedRule[] }` — Use `parseDocument` from `yaml` package. Get `doc.get('rules', true) as YAMLSeq`. Iterate items, extract Scalar values with `isScalar()` check. Build ParsedRule array with commentBefore and comment from Scalar nodes. Return both the parsed Document (for later serialization) and the rules array.

3. `groupByProxyGroup(rules: ParsedRule[]): RuleBlock[]` — Group rules by `rule.target`. Each unique target becomes a block. Block name = target name. Block rules ordered as they appear in original. Block id = `block-pg-{index}`.

4. `groupBySections(rules: ParsedRule[]): RuleBlock[]` — Scan commentBefore of each rule for section patterns: `# >>> NAME`, `# --- NAME ---`, `# <<< NAME`, emoji-prefixed lines like `# ⚪🔵🔴 RU сайты`. Use regex patterns from research. Rules before any section marker go into "Без секции" block. Block id = `block-sec-{index}`.

5. `groupTwoLevel(rules: ParsedRule[]): RuleBlock[]` — First group by sections, then within each section sub-group by proxy-group. Block name = "Section / ProxyGroup". Block id = `block-2l-{index}`.

6. `serializeRulesToConfig(doc: Document, blocks: RuleBlock[]): string` — Flatten blocks back to ordered rules. Rebuild YAMLSeq items array preserving commentBefore and commentInline on each Scalar node. Use `doc.createNode(rule.raw)` to create new Scalars, attach comments. Return `doc.toString()`.

7. `flattenBlocksToRules(blocks: RuleBlock[]): ParsedRule[]` — Simple utility: `blocks.flatMap(b => b.rules)`.

8. `buildRuleRaw(type: string, value: string, target: string, noResolve: boolean): string` — Reconstruct rule string from components. Handle MATCH (no value), compound rules (value already contains parens), standard rules.

**Import convention:** `import { parseDocument, YAMLSeq, Scalar, isScalar } from 'yaml'` — clearly distinct from js-yaml used in ConfigEditor.

Export all types and functions.

**Step 1:** Create `src/stores/rules-editor.ts`:

Zustand store wrapped with `temporal` middleware from `zundo` for undo/redo.

**State interface:**
```typescript
interface RulesEditorState {
  // Data
  blocks: RuleBlock[]
  originalYaml: string        // for diff preview
  currentYaml: string         // serialized current state for diff
  proxyGroups: string[]       // list of available proxy-group names (for dropdowns)

  // UI state (not tracked by undo)
  dirty: boolean
  changeCount: number         // for Apply badge counter
  loading: boolean
  error: string | null

  // Actions
  loadRules: (configYaml: string) => void
  reorderBlocks: (oldIndex: number, newIndex: number) => void
  reorderRules: (blockId: string, oldIndex: number, newIndex: number) => void
  addRule: (blockId: string, type: string, value: string, target: string, noResolve: boolean) => void
  removeRule: (blockId: string, ruleId: string) => void
  changeBlockTarget: (blockId: string, newTarget: string) => void
  changeRuleTarget: (blockId: string, ruleId: string, newTarget: string) => void
  createBlock: (name: string, target: string) => void
  removeBlock: (blockId: string) => void
  resetChanges: () => void
  markSaved: () => void
  serialize: () => string      // get current YAML string
}
```

**Implementation details:**
- `temporal` middleware wraps the store. Use `partialize` option to only track `blocks` changes (not UI state like `dirty`, `loading`, `error`).
- `loadRules`: calls `parseRulesFromConfig` + current grouping mode from settings store → sets blocks, originalYaml, proxyGroups (extracted from all unique targets). Stores the parsed Document in a module-level variable (not in Zustand — Document objects are not serializable). Resets dirty/changeCount.
- All mutation actions (reorder, add, remove, change target, create/remove block): update `blocks`, increment `changeCount`, set `dirty: true`. After each mutation, re-serialize via `serializeRulesToConfig` to keep `currentYaml` updated for diff.
- `reorderBlocks`: uses `arrayMove` pattern — splice old index, insert at new index.
- `reorderRules`: same arrayMove inside the specific block's rules array.
- `addRule`: creates new ParsedRule with `buildRuleRaw`, appends to block.rules.
- `removeRule`: filters out rule from block.rules.
- `changeBlockTarget`: updates all rules in block to new target (rebuilds raw strings). Updates block.target.
- `changeRuleTarget`: updates single rule target and raw string.
- `createBlock`: creates new RuleBlock with empty rules, adds to end of blocks.
- `removeBlock`: filters out block.
- `resetChanges`: reloads from originalYaml (re-parses, re-groups).
- `markSaved`: updates originalYaml to currentYaml, resets dirty/changeCount.
- `serialize`: calls `serializeRulesToConfig` with stored Document and current blocks.

**Undo/redo access pattern:**
```typescript
// Consumers use:
const { undo, redo, pastStates, futureStates } = useRulesEditorStore.temporal.getState()
```

Store is volatile (no persist) — rules loaded fresh from config on page visit.
