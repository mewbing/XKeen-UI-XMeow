```python
XKEEN_INIT = os.environ.get('XKEEN_INIT_SCRIPT', '/opt/etc/init.d/S24xray')
```

1. **POST /api/service/&lt;action&gt;** (action = start|stop|restart):
   - Validate action is one of ('start', 'stop', 'restart'), return 400 if not
   - Run `subprocess.run([XKEEN_INIT, action], capture_output=True, text=True, timeout=30)`
   - On success (returncode 0): return `{"status": "ok"}`, 200
   - On failure: return `{"error": stderr or "Command failed"}`, 500
   - On timeout: return `{"error": "Command timed out"}`, 500

2. **GET /api/service/status**:
   - Run `subprocess.run(['pidof', 'mihomo'], capture_output=True, text=True, timeout=5)`
   - If returncode 0: `{"running": true, "pid": int(stdout.strip())}`
   - If returncode != 0: `{"running": false, "pid": null}`
   - On exception: `{"running": false, "pid": null, "error": str(exc)}`

3. **GET /api/versions**:
   - xkeen version: try `subprocess.run(['xkeen', '-v'], capture_output=True, text=True, timeout=5)` and parse first line of stdout; on failure return "unknown"
   - dashboard version: hardcode "0.1.0" (will be updated in Phase 10 self-update)
   - Return `{"xkeen": "...", "dashboard": "0.1.0"}`

Place all new endpoints in a new section "Service Management" between the Health and Config API sections. Use try/except around all subprocess calls. Follow existing error handling pattern (app.logger.error + jsonify).

**Step 2: Create src/lib/mihomo-api.ts**
Mihomo REST API client. Uses `useSettingsStore.getState()` for URL and secret (same pattern as api.ts but for runtime API calls, not wizard).

Functions:
- `fetchMihomoVersion(): Promise<{version: string; meta: boolean}>` -- GET /version with Bearer auth, 5s timeout
- `upgradeCore(channel?: string): Promise<{status: string}>` -- POST /upgrade (or /upgrade?channel=xxx), Bearer auth, 120s timeout (long operation). Throw Error on non-ok response.
- `restartMihomo(): Promise<void>` -- POST /restart, Bearer auth, 10s timeout
- `fetchConnectionsSnapshot(): Promise<{downloadTotal: number; uploadTotal: number; connections: unknown[]}>` -- GET /connections, Bearer auth, 5s timeout (needed for active connections count on overview)

Helper functions (not exported):
- `getHeaders()` -- returns Record<string,string> with Authorization Bearer if secret exists
- `getBaseUrl()` -- returns mihomoApiUrl from settings store

**Step 3: Create src/lib/config-api.ts**
Config API client for service management. Uses `useSettingsStore.getState().configApiUrl` for base URL.

Types:
- `export type ServiceAction = 'start' | 'stop' | 'restart'`

Functions:
- `serviceAction(action: ServiceAction): Promise<void>` -- POST /api/service/{action}, 30s timeout. Throw Error with error message from response on failure.
- `fetchServiceStatus(): Promise<{running: boolean; pid: number | null}>` -- GET /api/service/status, 5s timeout
- `fetchVersions(): Promise<{xkeen: string; dashboard: string}>` -- GET /api/versions, 5s timeout

Helper: `getBaseUrl()` from settings store.

**Step 4: Create src/lib/format.ts**
Formatting utilities for overview metrics display.

Functions:
- `formatBytes(bytes: number): string` -- Convert bytes to human-readable (0 B, 1.2 KB, 3.4 MB, 1.1 GB, etc). Use 1024-based units: B, KB, MB, GB, TB.
- `formatSpeed(bytesPerSec: number): string` -- Returns `formatBytes(bytesPerSec) + '/s'`
- `formatUptime(startTime: number | null): string` -- Returns '--' if null. Otherwise computes diff from Date.now() in seconds. Format: "Xd Xh Xm" if days > 0, "Xh Xm Xs" if hours > 0, "Xm Xs" if minutes > 0, "Xs" otherwise.

All functions are pure (no side effects, no imports besides standard lib).
