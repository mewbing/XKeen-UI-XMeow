**Create src/hooks/use-mihomo-ws.ts:**

Generic WebSocket hook for mihomo streaming endpoints with auto-reconnect.

```typescript
function useMihomoWs<T>(
  path: string,            // "/traffic", "/memory", "/connections"
  onMessage: (data: T) => void,
  interval?: number        // for /connections?interval=
): void
```

Implementation details:
- Read `mihomoApiUrl` and `mihomoSecret` from `useSettingsStore` with selective subscriptions
- Convert HTTP URL to WS: replace `^http` with `ws`
- Build URL with query params: `token` (if secret exists), `interval` (if provided)
- Store WebSocket instance in `useRef<WebSocket | null>`
- `connect()` function wrapped in `useCallback`:
  - Create WebSocket with full URL
  - `ws.onmessage`: parse JSON, call `onMessage(data as T)`. Wrap in try/catch, ignore parse errors.
  - `ws.onclose`: auto-reconnect after 3 seconds via `setTimeout(() => connect(), 3000)`. Store timeout id in ref and clear on cleanup.
  - `ws.onerror`: call `ws.close()` (triggers onclose -> reconnect)
- `useEffect` calls `connect()`, cleanup closes WS and clears reconnect timeout
- Use `useRef` for `onMessage` callback to avoid stale closures (update ref on every render, use ref.current in ws.onmessage). This prevents re-creating WebSocket on every callback change.

**Create src/stores/overview.ts:**

Zustand store for overview metrics state + traffic history rolling buffer.

```typescript
const HISTORY_LENGTH = 60  // 60 seconds of data

interface TrafficPoint {
  time: number  // sequential counter (0..59)
  up: number    // bytes/sec
  down: number  // bytes/sec
}

interface OverviewState {
  // Current values
  uploadSpeed: number
  downloadSpeed: number
  uploadTotal: number
  downloadTotal: number
  memoryInuse: number
  activeConnections: number

  // Display mode
  metricsMode: 'compact' | 'panels'

  // History for chart
  trafficHistory: TrafficPoint[]

  // Uptime tracking (client-side)
  startTime: number | null

  // Versions
  mihomoVersion: string
  dashboardVersion: string
  xkeenVersion: string

  // Actions
  updateTraffic: (data: { up: number; down: number; upTotal: number; downTotal: number }) => void
  updateMemory: (data: { inuse: number }) => void
  updateConnections: (count: number) => void
  setStartTime: (time: number) => void
  setMetricsMode: (mode: 'compact' | 'panels') => void
  setVersions: (v: { mihomo?: string; dashboard?: string; xkeen?: string }) => void
}
```

Key implementation notes:
- Initialize `trafficHistory` with 60 zero-value points (time: 0..59)
- `updateTraffic`: append new point with `time: Date.now()`, slice to keep last `HISTORY_LENGTH` entries. Update current speed and total values in single `set()` call (batch to prevent multiple re-renders).
- `updateMemory`: set `memoryInuse` only
- `updateConnections`: set `activeConnections` only
- `setMetricsMode`: toggle between 'compact' and 'panels'
- `setVersions`: merge partial version update
- Do NOT persist this store (volatile real-time data, no localStorage)
