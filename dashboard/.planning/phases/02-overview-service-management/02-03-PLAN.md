**Create src/hooks/use-service-status.ts:**

Custom hook that polls service status from Config API.

```typescript
export function useServiceStatus(intervalMs: number = 5000): {
  running: boolean
  pid: number | null
  loading: boolean
  error: string | null
  refresh: () => void
}
```

Implementation:
- State: `running` (boolean), `pid` (number|null), `loading` (boolean), `error` (string|null)
- On mount: fetch immediately, then setInterval with `intervalMs`
- Call `fetchServiceStatus()` from config-api.ts
- On success: update running/pid, clear error, set loading false
- On error: set error message, set loading false, keep last known running state
- `refresh()`: manually trigger an immediate fetch (useful after service action)
- Cleanup: clear interval on unmount
- After a service action (start/stop/restart), caller should call `refresh()` with a small delay (2-3 seconds) to allow service to start/stop

**Create src/components/overview/ServiceControl.tsx:**

Per user decision: one button with context dropdown-menu (Start/Stop/Restart) + AlertDialog confirmation.

Structure:
- Uses shadcn DropdownMenu with DropdownMenuTrigger (a Button), DropdownMenuContent, DropdownMenuItems
- Button shows current status: green circle + "Запущен" or red circle + "Остановлен" (reads `running` from useServiceStatus)
- Three menu items: "Запустить" (Play icon), "Остановить" (Square icon), "Перезапустить" (RotateCcw icon)
- "Запустить" only enabled when stopped; "Остановить" and "Перезапустить" only enabled when running
- Each destructive action (Stop, Restart) wraps its DropdownMenuItem inside AlertDialog:
  - Title: "Остановить сервис?" / "Перезапустить сервис?"
  - Description: "Все активные подключения будут закрыты." (for stop) / "Сервис будет временно недоступен." (for restart)
  - Cancel: "Отмена", Action: "Остановить" / "Перезапустить"
  - Use `onSelect={(e) => e.preventDefault()}` on DropdownMenuItem to prevent dropdown from closing before dialog opens
- Start action does not need confirmation (non-destructive)
- On action: call `serviceAction(action)` from config-api.ts. Show loading state on button (spinner). On success, set optimistic status ("Restarting..." / "Stopping...") then call `refresh()` after 3 seconds. On error, show error via inline message or console.
