Create the SSH server infrastructure for accepting reverse tunnel connections from remote agents, and the token-based authentication store for agent management.

Purpose: This is the backend foundation for remote management. The token store provides CRUD operations for agent token management with JSON file persistence.

### server.go — SSH server lifecycle

**Server struct:**
```go
type Server struct {
    mu             sync.RWMutex
    config         *ssh.ServerConfig
    listener       net.Listener
    agents         map[string]*AgentConn // agentID -> agent
    done           chan struct{}
    onAgentChange  func() // callback for notifying WebSocket subscribers
    validateToken  func(token string) (agentID string, ok bool)
    updateLastSeen func(agentID string)
}
```

**NewServer(hostKeyPath string, validateToken func, updateLastSeen func) (*Server, error):**
- Load or generate ed25519 host key (call loadOrGenerateHostKey)
- Create ssh.ServerConfig with PasswordCallback that validates token
- Token is sent as SSH password by agent, username is device_name (informational)
- On successful auth, store agentID in Permissions.Extensions["agent-id"]

**loadOrGenerateHostKey(path string) (ssh.Signer, error):**
- Try os.ReadFile(path) -> ssh.ParsePrivateKey
- If not found: ed25519.GenerateKey -> x509.MarshalPKCS8PrivateKey -> pem.Encode -> os.WriteFile(path, 0600) -> ssh.ParsePrivateKey
- Host key path: e.g., `/opt/etc/xmeow-ui/ssh_host_ed25519_key`

**ListenAndServe(addr string) error:**
- net.Listen("tcp", addr)
- Accept loop in goroutine, each connection -> go handleConnection()
- Respect s.done channel for graceful shutdown

**handleConnection(netConn net.Conn):**
- ssh.NewServerConn(netConn, s.config)
- Extract agentID from Permissions.Extensions
- Create AgentConn, register with s.registerAgent()
- Defer s.unregisterAgent() — MUST close all tunnel listeners + notify UI
- go handleGlobalRequests() for tcpip-forward and heartbeat
- Range over chans, reject all channel opens from agent (Prohibited)
- When chans closes (connection dropped), the defer runs cleanup

**registerAgent(id string, agent *AgentConn) / unregisterAgent(id string):**
- Lock mu, add/remove from agents map
- Call onAgentChange() callback to notify WebSocket subscribers
- On unregister: call agent.Close() to clean up all listeners

**GetAgent(id string) *AgentConn** — thread-safe lookup
**GetAllAgents() []AgentInfo** — returns list of all connected agents as AgentInfo

**SetOnAgentChange(fn func())** — sets callback for status updates

**Shutdown():**
- Close done channel
- Close listener
- Close all agent connections

### tunnel.go — Reverse tunnel port forwarding

**SSH RFC 4254 types:**
```go
type forwardRequest struct { BindAddr string; BindPort uint32 }
type forwardResponse struct { Port uint32 }
type forwardedTCPPayload struct { Addr string; Port uint32; OriginAddr string; OriginPort uint32 }
```

**handleGlobalRequests(agentID string, agent *AgentConn, conn ssh.Conn, reqs <-chan *ssh.Request):**
- Switch on req.Type:
  - "tcpip-forward" -> handleTCPIPForward(agent, req)
  - "cancel-tcpip-forward" -> close corresponding listener, Reply(true, nil)
  - "heartbeat" -> unmarshal HeartbeatData JSON, update agent fields, call updateLastSeen(agentID), Reply(true, nil)
  - default -> Reply(false, nil)

**handleTCPIPForward(agent *AgentConn, req *ssh.Request):**
- Unmarshal forwardRequest from payload
- net.Listen("tcp", "127.0.0.1:0") — dynamic port allocation
- Store mapping: agent.tunnelPorts[remotePort] = localPort
- Track listener in agent.listeners for cleanup
- Reply(true, forwardResponse{Port: localPort})
- Go routine: accept loop, for each connection -> go forwardConnection()

**forwardConnection(agent *AgentConn, local net.Conn, remoteAddr string, remotePort uint32):**
- Build forwardedTCPPayload with remote addr/port
- agent.conn.OpenChannel("forwarded-tcpip", payload) — opens channel to agent
- go ssh.DiscardRequests(reqs) on channel requests
- Bidirectional io.Copy between local conn and SSH channel
- Proper cleanup with deferred Close() on both sides

CRITICAL: All SSH channel consumers (chans, reqs) MUST be drained to avoid deadlock. Use `go ssh.DiscardRequests()` for unused request channels. Always range over the chans channel even if rejecting all.

CRITICAL: Dynamic port allocation with `net.Listen("tcp", "127.0.0.1:0")` — never hardcode ports to avoid conflicts between multiple agents.

CRITICAL: Cleanup on disconnect — close ALL listeners and their accepted connections when agent disconnects. This prevents port leaks.
