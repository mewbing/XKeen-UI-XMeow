Purpose: This plan connects the backend infrastructure (Plan 01) to the frontend by creating API endpoints that the UI can call. It also integrates the SSH server into the main server lifecycle so it starts/stops alongside the HTTP server.

Output: REST API at /api/remote/*, WebSocket at /ws/remote/status, HTTP proxy at /api/remote/{agentID}/proxy/* and /api/remote/{agentID}/mihomo/*, SSH server integrated into main server startup.

**2. Create internal/remote/proxy.go:**

Create an HTTP proxy handler that forwards requests through SSH tunnel to remote agent services.

```go
package remote

import (
    "fmt"
    "net/http"
    "net/http/httputil"
    "net/url"
    "strings"

    "github.com/go-chi/chi/v5"
    "github.com/mewbing/XKeen-UI-XMeow/internal/sshserver"
)

type Proxy struct {
    sshServer *sshserver.Server
}

func NewProxy(sshServer *sshserver.Server) *Proxy {
    return &Proxy{sshServer: sshServer}
}
```

**ProxyToAgent(w http.ResponseWriter, r *http.Request):**
- Extract agentID from chi.URLParam(r, "agentID")
- Get agent from sshServer.GetAgent(agentID)
- If nil -> 502 Bad Gateway JSON: `{"error": "agent not connected"}`
- Get local forwarded port for XMeow API: agent.GetLocalPort(5000)
- If not found -> 502: `{"error": "tunnel not established for port 5000"}`
- Create httputil.ReverseProxy with Rewrite:
  - Set target URL to `http://127.0.0.1:{localPort}`
  - Strip prefix `/api/remote/{agentID}/proxy` from path
- proxy.ServeHTTP(w, r)

**ProxyToAgentMihomo(w http.ResponseWriter, r *http.Request):**
- Same as above but for port 9090 (mihomo API)
- Strip prefix `/api/remote/{agentID}/mihomo`
- This enables the frontend to access the remote router's mihomo API

Use `httputil.ReverseProxy` with `Rewrite` function (NOT `Director`) — per project convention from Phase 12.

**Error responses** should be JSON with Content-Type application/json, matching the existing API pattern.

**REST Endpoints:**

`ListAgents(w, r)` — GET /api/remote/agents
- Get registered tokens from store.List()
- Get online agents from sshServer.GetAllAgents()
- Merge: for each token, check if matching agent is online (by ID)
- Return JSON array of agent objects with fields: id, name, online, arch, mihomo_ver, ip, uptime_sec, last_heartbeat, created_at

`CreateToken(w, r)` — POST /api/remote/tokens
- Read JSON body: `{ "name": "Keenetic-Office" }`
- Call store.Create(name)
- Return JSON: full token (this is the ONLY time the full token is visible)
- Return 201 Created

`ListTokens(w, r)` — GET /api/remote/tokens
- Return store.List() (tokens masked — first 8 chars + "...")

`RevokeToken(w, r)` — DELETE /api/remote/tokens/{id}
- Extract id from chi.URLParam
- Call store.Revoke(id)
- Return 204 No Content

`DeleteToken(w, r)` — DELETE /api/remote/agents/{id}
- Extract id from chi.URLParam
- Call store.Delete(id) — removes token entirely
- Return 204 No Content

`ProxyToAgent(w, r)` — handled by proxy.ProxyToAgent
`ProxyToAgentMihomo(w, r)` — handled by proxy.ProxyToAgentMihomo

**WebSocket endpoint — /ws/remote/status:**

`WsRemoteStatus` handler:
- Upgrade to WebSocket (gorilla/websocket)
- Auth check: read Authorization header or "token" query param (same pattern as ws_terminal.go)
- On connect: send current agent list as JSON
- Register for agent change notifications via sshServer.SetOnAgentChange callback
- When agent status changes: push updated list to all connected WebSocket clients
- Heartbeat: ping every 30s to keep connection alive
- Use a simple broadcast pattern: maintain a list of connected WS clients (sync.Mutex + slice of *websocket.Conn)

Pattern from existing ws_terminal.go for auth check:

if token == "" {
