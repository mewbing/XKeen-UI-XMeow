<objective>
Create the xmeow-agent binary for remote routers and extend setup.sh with agent installation support.

Purpose: The agent is a lightweight Go binary that runs on remote routers behind NAT. It maintains a persistent SSH reverse tunnel to the master server, enabling the master to proxy API requests to the remote router. It also sends periodic heartbeat data so the master knows the agent's status. The setup.sh extension makes installation a one-liner.

- Username: device_name (informational)
- Password: 64-char hex token

Agent requests reverse tunnel forwarding:
- SSH global request type: "tcpip-forward"
- Payload: { BindAddr: "0.0.0.0", BindPort: uint32 } for ports 5000, 9090, 22

Agent sends heartbeat:
- SSH global request type: "heartbeat" (wantReply: true)

```go
type HeartbeatData struct {

}
```

  <action>
Create `cmd/xmeow-agent/main.go` — a SINGLE FILE agent binary. Keep it minimal for small binary size.

CRITICAL: Do NOT import any packages beyond what's strictly needed. Avoid: goccy/go-yaml, gorilla/websocket, chi, any HTTP packages, the internal/ packages from xmeow-server. The agent must be a standalone binary with minimal dependencies. Target binary size: < 4MB compressed.

**Imports allowed:**
- `golang.org/x/crypto/ssh` — SSH client
- `encoding/json` — config and heartbeat
- `crypto/rand`, `crypto/ed25519` — not needed (agent is client only)
- `os`, `os/exec`, `os/signal` — system operations
- `net`, `io` — network
- `fmt`, `log`, `flag`, `time`, `math`, `math/rand`, `runtime`, `strings`, `syscall`

**Config struct:**
```go
type Config struct {
    ServerHost string `json:"server_host"`
    ServerPort int    `json:"server_port"`
    Token      string `json:"token"`
    DeviceName string `json:"device_name"`
}
```

**Version variable:** `var Version = "0.1.0"` with ldflags support.

**main():**
1. Parse flags: `-config` (path to agent.conf, default `/opt/etc/xmeow-ui/agent.conf`), `--version`/`-v` (print version, exit)
2. Load config from JSON file
3. Set up signal handler (SIGINT, SIGTERM) for graceful shutdown
4. Enter connectLoop()

**connectLoop():**
- Infinite loop with exponential backoff on failure
- Call connect() -> if error, backoff(attempt) with jitter, sleep, retry
- On successful connect, reset attempt counter
- Call runSession() which blocks until disconnect
- On disconnect, log and loop (will reconnect)
- Respect done channel for shutdown

**backoff(attempt int) time.Duration:**
- Initial: 1s, max: 5min, jitter: +/-30%
- Formula: min(initialDelay * 2^(attempt-1), maxDelay) + jitter

**connect() (*ssh.Client, error):**
- Build ssh.ClientConfig:
  - User: config.DeviceName
  - Auth: ssh.Password(config.Token)
  - HostKeyCallback: ssh.InsecureIgnoreHostKey() (per project pattern from Phase 17 — home router local network)
  - Timeout: 10s
- ssh.Dial("tcp", fmt.Sprintf("%s:%d", config.ServerHost, config.ServerPort), clientConfig)

**runSession(client *ssh.Client):**
1. Request reverse port forwarding for ports [5000, 9090, 22]:
   - For each port: send "tcpip-forward" global request with payload {BindAddr: "0.0.0.0", BindPort: port}
   - Log success/failure for each
2. Start heartbeat goroutine
3. Accept forwarded connections:
   - client.HandleChannelOpen("forwarded-tcpip") to receive incoming channel opens
   - For each channel: parse the forwardedTCPPayload to determine target port
   - Dial locally to `127.0.0.1:{targetPort}` and bidirectional copy
4. Block until client disconnects or done signal

**heartbeatLoop(conn ssh.Conn):**
- Every 30 seconds, collect heartbeat data:
  - DeviceName: from config
  - Arch: runtime.GOARCH
  - MihomoVer: exec.Command("mihomo", "-v") and parse first line (or "unknown" on error)
  - Uptime: time.Since(startTime).Seconds()
  - IP: extract outbound IP by dialing UDP to 8.8.8.8:80 and reading LocalAddr
- JSON marshal -> conn.SendRequest("heartbeat", true, payload)
- If error or !ok -> return (connection lost, connectLoop will handle reconnect)

**handleForwardedChannels(client *ssh.Client):**
- Range over client.HandleChannelOpen("forwarded-tcpip")
- For each new channel: parse forwardedTCPPayload from ExtraData
- Determine target local port from Payload.Port
- net.Dial("tcp", fmt.Sprintf("127.0.0.1:%d", targetPort))
- Accept the channel, bidirectional io.Copy between SSH channel and local connection
- Cleanup with deferred Close on both

IMPORTANT: The `HandleChannelOpen("forwarded-tcpip")` returns a channel that must be consumed. When SSH connection drops, this channel closes and the function returns.

IMPORTANT: Handle `os/exec.Command("mihomo", "-v")` errors gracefully — return "unknown" if mihomo is not installed or fails.

3. **Agent installation function `install_agent()`:**
   - Reuse existing arch detection (already in setup.sh)
   - Download `xmeow-agent-linux-{arch}` from GitHub releases (same release tag as server)
   - Extract tar.gz, copy binary to `/opt/etc/xmeow-ui/xmeow-agent`
   - Set permissions: `chmod +x`
   - Create agent.conf template if not exists:
     ```json
     {
       "server_host": "",
       "server_port": 2222,
       "token": "",
       "device_name": ""
     }
     ```
   - Interactive prompts (if running interactively, not piped):
     - "Адрес мастер-сервера (IP или домен):" -> server_host
     - "Порт SSH мастер-сервера [2222]:" -> server_port (default 2222)
     - "Токен агента:" -> token
     - "Имя устройства:" -> device_name (default: hostname)
   - Write agent.conf with provided values
   - Create init.d script `/opt/etc/init.d/S99xmeow-agent`:
     ```sh
     #!/bin/sh
     CONF="/opt/etc/xmeow-ui/agent.conf"
     BIN="/opt/etc/xmeow-ui/xmeow-agent"
     PIDFILE="/opt/var/run/xmeow-agent.pid"

     case "$1" in
       start)
         if [ -f "$PIDFILE" ] && kill -0 "$(cat "$PIDFILE")" 2>/dev/null; then
           echo "xmeow-agent already running"
         else
           echo "Starting xmeow-agent..."
           "$BIN" -config "$CONF" > /dev/null 2>&1 &
           echo $! > "$PIDFILE"
         fi
         ;;
       stop)
         if [ -f "$PIDFILE" ]; then
           kill "$(cat "$PIDFILE")" 2>/dev/null
           rm -f "$PIDFILE"
         fi
         ;;
       restart) "$0" stop; sleep 1; "$0" start ;;
       status)
         if [ -f "$PIDFILE" ] && kill -0 "$(cat "$PIDFILE")" 2>/dev/null; then
           echo "xmeow-agent running (PID $(cat "$PIDFILE"))"
         else
           echo "xmeow-agent not running"
         fi
         ;;
     esac
     ```
   - `chmod +x` on init.d script
   - Start the agent service
   - Print success message with instructions
