**1. Create `src/components/terminal/TerminalView.tsx`:**

Core terminal component. Manages xterm.js instance lifetime and WS integration.

Key architectural decision: The Terminal instance and addons live in refs, NOT state. The xterm.js div is kept mounted (via CSS `display:none` when modal closed) so the Terminal instance persists across open/close cycles. However, since this component is rendered inside a Dialog, we need to handle mounting carefully.

**Strategy for session persistence:** The Terminal instance is created once on first mount and stored in a ref. The container div is always present when modal is open. When modal closes, Dialog unmounts children. To persist the terminal across modal open/close:
- Store the Terminal instance in a module-level variable (outside React) OR in the Zustand store as a non-serializable ref
- **Best approach:** Keep the terminal div mounted in AppLayout (always rendered, hidden when modal closed). The Dialog just controls visibility. This avoids Dialog unmounting issues entirely.

**Actually, simplest approach per CONTEXT.md:** "Session persists when modal is closed -- reopening restores the active session." Use this strategy:
- TerminalView is rendered in AppLayout (outside Dialog), positioned absolutely, hidden via `hidden` attribute when modal is closed
- The Dialog/modal only provides the overlay backdrop, not the terminal content
- OR: Use Radix Dialog with `forceMount` on DialogContent to prevent unmount

**Use approach: Render TerminalView directly inside a custom modal wrapper that does NOT unmount.** The modal wrapper uses fixed positioning + z-index for overlay, and `display:none`/`display:flex` to show/hide. Do NOT use Radix Dialog (it unmounts content).

**Terminal initialization (useEffect, runs once):**
1. If `initializedRef.current` or no containerRef, return
2. Create Terminal with: `cursorBlink: true`, `fontSize` from store, `fontFamily: "'JetBrains Mono', 'Cascadia Code', 'Fira Code', Consolas, monospace'`, `scrollback: 1000`, `theme: TERMINAL_THEME`, `allowProposedApi: true` (needed by some addons)
3. Load FitAddon, SearchAddon, WebLinksAddon
4. `term.open(containerRef.current)`
5. `fitAddon.fit()`
6. Set up ResizeObserver on container: call `fitAddon.fit()` on resize, then send new dimensions to backend via WS resize()
7. Terminal `onData` handler: send input to WS via `sendInput(data)` -- xterm.js onData already provides the string
8. Set `initializedRef.current = true`
9. Write welcome message: `term.writeln('\\x1b[90mXMeow Terminal\\x1b[0m')` and `term.writeln('\\x1b[90mНажмите Connect для подключения к роутеру\\x1b[0m')`

**WS integration via useTerminalWs:**
```typescript
const { wsConnected, connect, sendInput, resize, disconnect } = useTerminalWs({
  enabled: true, // always enabled -- WS connects on mount, SSH connects on user action
  onData: (data) => termRef.current?.write(data),
  onConnected: (reused) => {
    useTerminalStore.getState().setConnected(true)
    useTerminalStore.getState().setConnecting(false)
    if (reused) toast.info('Восстановлено подключение к существующей сессии')
  },
  onDisconnected: (reason) => {
    useTerminalStore.getState().setConnected(false)
    useTerminalStore.getState().setSessionAlive(false)
    termRef.current?.writeln(`\\r\\n\\x1b[91mОтключено: ${reason}\\x1b[0m`)
  },
  onError: (message) => {
    useTerminalStore.getState().setConnecting(false)
    useTerminalStore.getState().setError(message)
    termRef.current?.writeln(`\\r\\n\\x1b[91mОшибка: ${message}\\x1b[0m`)
    toast.error(message)
  },
})
```

**Font size changes:** Subscribe to terminal store fontSize. When it changes, call `termRef.current.options.fontSize = newSize` then `fitRef.current.fit()`.
