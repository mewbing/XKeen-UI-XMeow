**1. Create `src/hooks/useTerminalWs.ts`:**

WebSocket hook for terminal with hybrid binary/JSON protocol. Follow `useLogWebSocket.ts` patterns closely.

```typescript
/**
 * WebSocket hook for terminal SSH bridge.
 *
 * Connects to /ws/terminal on the Go backend.
 * Protocol: binary frames for terminal I/O, JSON text frames for control.
 * Auth via query parameter token=SECRET.
 * Auto-reconnects on disconnect (1s delay).
 * Ping/pong keepalive every 30s.
 * 150ms startup delay for React Strict Mode safety.
 */
```

Interface:
```typescript
interface UseTerminalWsOptions {
  enabled: boolean
  onData: (data: Uint8Array) => void       // binary terminal output
  onConnected: (reused?: boolean) => void   // SSH connected
  onDisconnected: (reason: string) => void  // SSH disconnected
  onError: (message: string) => void        // SSH or WS error
}

interface UseTerminalWsReturn {
  wsConnected: boolean          // WS layer connected
  connect: (host: string, port: number, user: string, password: string, cols: number, rows: number) => void
  sendInput: (data: string | Uint8Array) => void
  resize: (cols: number, rows: number) => void
  disconnect: () => void
}
```

Implementation details:

`getTerminalWsUrl()`:
- Read `configApiUrl` from settings store
- Replace `http` with `ws` in base URL
- Append `/ws/terminal`
- Append `?token=` + mihomoSecret from settings store
- If no configApiUrl: use `location.protocol`/`location.host` pattern (same as useLogWebSocket)

`useTerminalWs(options)`:
- Same ref-based callback pattern as useLogWebSocket (enabledRef, onDataRef, etc.)
- Same 150ms startup delay, disposed flag, reconnect on close (1s delay)
- Ping every 30s via setInterval

**WebSocket message handling:**

`ws.onmessage = (event) => {`
- Check `typeof event.data`:
  - If `Blob` (binary from server): convert to Uint8Array via `event.data.arrayBuffer()` then call `onDataRef.current(new Uint8Array(buffer))`
  - If `string` (JSON control message): parse JSON, handle:
    - `"connected"`: call `onConnectedRef.current(data.reused)`
    - `"disconnected"`: call `onDisconnectedRef.current(data.reason || 'server')`
    - `"error"`: call `onErrorRef.current(data.message)`
    - `"pong"`: ignore (keepalive response)

**Sending:**

`connect()`: send JSON text frame `{ type: 'connect', host, port, user, password, cols, rows }`
`sendInput(data)`:
- If string: encode to Uint8Array via `new TextEncoder().encode(data)`
- Send as binary: `ws.send(data)` where data is Uint8Array
`resize(cols, rows)`: send JSON `{ type: 'resize', cols, rows }`
`disconnect()`: send JSON `{ type: 'disconnect' }`

Important: set `ws.binaryType = 'blob'` after creating the WebSocket (this is the default in browsers, but be explicit). Alternatively, set `ws.binaryType = 'arraybuffer'` and handle ArrayBuffer directly -- this avoids the async Blob.arrayBuffer() call. **Use `'arraybuffer'`** -- simpler:

```typescript
ws.binaryType = 'arraybuffer'
// ...
ws.onmessage = (event) => {
  if (event.data instanceof ArrayBuffer) {
    onDataRef.current(new Uint8Array(event.data))
  } else {
    // string -- JSON control message
    const msg = JSON.parse(event.data)
    // ...
  }
}
```

Return: `{ wsConnected, connect, sendInput, resize, disconnect }`

All return functions wrapped in `useCallback`.
