**2. Create `internal/terminal/session.go`:**

Package `terminal`. Implements SSH client session with PTY.

```go
type Session struct {
    mu        sync.Mutex
    client    *ssh.Client
    session   *ssh.Session
    stdin     io.WriteCloser
    stdout    io.Reader
    lastInput time.Time
    done      chan struct{}
    closed    bool
}
```

Methods:
- `NewSession() *Session` -- creates session with `done` channel
- `Connect(host string, port int, user, password string, cols, rows int) error` -- establishes SSH connection:
  - `ssh.ClientConfig` with `ssh.Password(password)`, `ssh.InsecureIgnoreHostKey()` (per discretion decision -- home router localhost connection), 10s timeout
  - `ssh.Dial("tcp", host:port)`
  - `client.NewSession()` + `session.RequestPty("xterm-256color", rows, cols, modes)` with ECHO=1, ISPEED/OSPEED=14400
  - `session.StdinPipe()`, `session.StdoutPipe()`
  - `session.Shell()` -- uses user's default login shell from /etc/passwd (per research recommendation -- no shell detection needed)
  - Sets `lastInput = time.Now()`
  - On any error, clean up partial resources (close session/client)
- `Write(data []byte) (int, error)` -- writes to stdin, updates lastInput timestamp (mutex for lastInput only)
- `Read(buf []byte) (int, error)` -- reads from stdout (blocking, used in goroutine)
- `Resize(cols, rows int) error` -- calls `session.WindowChange(rows, cols)` (note: WindowChange takes rows first!)
- `Close()` -- idempotent close: stdin, session, client, close done channel. Protected by mutex and `closed` flag
- `Alive() bool` -- returns `!closed`
- `LastInput() time.Time` -- returns lastInput (mutex-protected)
- `Done() <-chan struct{}` -- returns done channel for select

**3. Create `internal/terminal/hub.go`:**

Package `terminal`. Single-session manager.

```go
type Hub struct {
    mu       sync.Mutex
    session  *Session
    clients  int       // number of attached WS clients
    timeout  time.Duration
    done     chan struct{}
}
```

Methods:
- `NewHub(timeout time.Duration) *Hub` -- default 30 minutes timeout
- `GetSession() *Session` -- returns current session if alive, nil otherwise
- `CreateSession() *Session` -- creates new session (closes old one if exists), stores in hub
- `AttachClient()` -- increments client count
- `DetachClient()` -- decrements client count
- `Shutdown()` -- closes session if alive, closes done channel

Start an inactivity checker goroutine in `NewHub`:
- Ticker every 1 minute
- If session alive and `time.Since(session.LastInput()) > timeout`: close session
- Select on both ticker and done channel for clean shutdown

Important: hub does NOT manage WS connections directly. It only tracks the SSH session and client count. The WS handler manages individual WS connections.

**1. Create `internal/handler/ws_terminal.go`:**

WebSocket handler with auth and binary/JSON message routing.

```go
type WsTerminalHandler struct {
    hub *terminal.Hub
    cfg *config.AppConfig
}

func NewWsTerminalHandler(hub *terminal.Hub, cfg *config.AppConfig) *WsTerminalHandler
```

`ServeHTTP(w http.ResponseWriter, r *http.Request)`:
1. **Auth check BEFORE upgrade** (unlike /ws/logs which has no auth):
   - Read token from `r.URL.Query().Get("token")`
   - Fallback: `strings.TrimPrefix(r.Header.Get("Authorization"), "Bearer ")`
   - Get secret via `config.GetMihomoSecret(h.cfg.MihomoConfigPath)`
   - If secret is not empty and token doesn't match: return 401 JSON error
   - If secret is empty: skip auth (same pattern as AuthMiddleware)
2. **Upgrade** to WebSocket using the shared `upgrader` from ws_logs.go
3. **Track client**: `hub.AttachClient()`, defer `hub.DetachClient()`
4. **Start output goroutine**: If hub has alive session, start a goroutine that reads from session.Read() in a loop and sends as `websocket.BinaryMessage` to WS conn. Use 4096-byte buffer. Exit on read error or session close.
5. **Read loop** (blocking, main goroutine):
   - `conn.ReadMessage()` -> check msgType
   - If `websocket.BinaryMessage`: terminal input -> `session.Write(data)`
   - If `websocket.TextMessage`: parse JSON control command:

Control message struct:
```go
type termCmd struct {
    Type     string `json:"type"`
    Host     string `json:"host,omitempty"`
    Port     int    `json:"port,omitempty"`
    User     string `json:"user,omitempty"`
    Password string `json:"password,omitempty"`
    Cols     int    `json:"cols,omitempty"`
    Rows     int    `json:"rows,omitempty"`
}
```

Handle commands:
- `"connect"`: Get or create session from hub. If session already alive, send `{"type":"connected","reused":true}`. Otherwise call `session.Connect(cmd.Host, cmd.Port, cmd.User, cmd.Password, cmd.Cols, cmd.Rows)`. On success: send `{"type":"connected"}` and start output goroutine. On error: send `{"type":"error","message":"..."}`.
- `"resize"`: Call `session.Resize(cmd.Cols, cmd.Rows)`. Silently ignore errors.
- `"disconnect"`: Close session. Send `{"type":"disconnected","reason":"user"}`.
- `"ping"`: Send `{"type":"pong"}`.

On WS close (read loop exits): stop output goroutine (use a `stopOutput` channel or context cancel), but do NOT close SSH session (it stays alive for reconnection within timeout window).

Helper function `sendJSON(conn *websocket.Conn, v interface{}) error` to write JSON text messages.

Helper function to start/stop the output reader goroutine safely. Use a context with cancel: when WS disconnects, cancel context to stop the read goroutine. The read goroutine should select between context.Done() and blocking session.Read(). Since session.Read() blocks, wrap it: read into channel from a goroutine, then select on channel vs context.
