  <action>
Create `src/components/versions/VersionsDialog.tsx` — the main dialog component:

**Props interface:**
```typescript
interface VersionsDialogProps {
  open: boolean
  defaultTab: string  // 'xkeen' | 'mihomo' | 'dashboard'
  onClose: () => void
}
```

**State:**
- `activeTab` (string) — controlled tab value, synced to `defaultTab` when dialog opens
- `confirmAction` (object | null) — shared AlertDialog state: `{ fn: () => void; title: string; description: string }`
- `overlayOpen` (boolean) — UpdateOverlay open state
- `overlayMode` ('server' | 'dist') — UpdateOverlay mode

**Key behaviors:**

1. **Tab sync on open** (Pitfall 6 prevention):
   ```typescript
   useEffect(() => { if (open) setActiveTab(defaultTab) }, [open, defaultTab])
   ```

2. **Close guard** (Pitfall 2):
   - Read `mihomoInstalling` from `useReleasesStore`
   - `handleClose`: if `mihomoInstalling` return early, else call `onClose()`
   - `onInteractOutside`: preventDefault if `mihomoInstalling`

3. **Tab lock during install** (Open Question 1 from RESEARCH):
   - Disable ALL non-active tab triggers when `mihomoInstalling` is true
   - Implementation: `disabled={mihomoInstalling && activeTab !== tabValue}` on each TabsTrigger

4. **Dialog structure:**
   ```
   Dialog > DialogContent (class: !max-w-2xl max-h-[85vh] flex flex-col)
     DialogHeader (shrink-0)
       DialogTitle: "Версии и обновления" (static, per RESEARCH recommendation)
     Tabs (value={activeTab} onValueChange={setActiveTab} class: flex-1 min-h-0 flex flex-col)
       TabsList (shrink-0)
         TabsTrigger value="xkeen": XKeen
         TabsTrigger value="mihomo": Mihomo
         TabsTrigger value="dashboard": Dashboard
       TabsContent value="xkeen" forceMount class="flex-1 min-h-0 data-[state=inactive]:hidden"
         XKeenTab active={activeTab === 'xkeen' && open} onClose={handleClose}
       TabsContent value="mihomo" forceMount class="flex-1 min-h-0 data-[state=inactive]:hidden"
         MihomoTab active={activeTab === 'mihomo' && open} onConfirm={setConfirmAction}
       TabsContent value="dashboard" forceMount class="flex-1 min-h-0 data-[state=inactive]:hidden"
         DashboardTab active={activeTab === 'dashboard' && open} onClose={handleClose} onConfirm={setConfirmAction} onOverlay={(mode) => { setOverlayMode(mode); setOverlayOpen(true) }}
   ```

5. **Shared AlertDialog** (Pattern 4 from RESEARCH):
   - Rendered as sibling of Dialog (separate portal)
   - `open={confirmAction !== null}`
   - Dynamic title/description from confirmAction state
   - On confirm: call `confirmAction.fn()`, set `confirmAction(null)`

6. **UpdateOverlay** (Pattern 3 from RESEARCH):
   - Rendered as sibling of Dialog: `<UpdateOverlay open={overlayOpen} mode={overlayMode} onClose={() => setOverlayOpen(false)} />`
   - DashboardTab's update/rollback handlers call `onClose()` (closing dialog) then trigger overlay via `onOverlay` callback

7. **Animations** (user decision): Tab content uses existing `animate-in fade-in-0 duration-200` inside each tab component (already handled in Task 1). No extra animation on TabsContent wrapper — the forceMount + hidden pattern means content is always there, just shown/hidden.

Export as named: `export function VersionsDialog({ open, defaultTab, onClose }: VersionsDialogProps)`
