<decisions>
## Implementation Decisions

### Обработка сломанных URL
- Если URL возвращает не-200: искать замену (альтернативный URL для того же набора правил)
- Если замену найти не удалось: закомментировать как `# BROKEN` и оставить для ручного разбора
- Проверка включает не только HTTP 200, но и непустой ответ (содержимое похоже на валидный набор правил)
- Проверять вручную (Claude проверяет каждый URL через curl/fetch), без написания отдельного скрипта

### Стратегия версий GitHub-ссылок
- Все GitHub URL (не только Anton111111) переводить на main/дефолтную ветку
- Перед переводом Anton111111 с тега на main: сначала сравнить структуру (вдруг пути/формат изменились)
- Для всех остальных GitHub-репозиториев: тоже переводить на main, если есть теги/коммиты в URL
- Проверить актуальность версий для всех GitHub-источников в конфиге

### Orphan-провайдеры
- Провайдеры без правил: подключить к соответствующей proxy-group (Claude определяет по имени и URL провайдера)
- Если провайдер явно не нужен (дубль, непонятное назначение): показать пользователю список сомнительных, он решит по каждому
- Проверять в обоих направлениях: orphan-провайдеры (определены, но не используются) И битые ссылки из rules на несуществующие провайдеры

### Пароль dashboard
- Пароль задаёт пользователь сам (не генерировать автоматически)
- Хранение в config.yaml открытым текстом — ок (локальный роутер)
- Смена пароля — отдельный шаг, НЕ в рамках автоматического выполнения фазы (оставить TODO-комментарий)

### Claude's Discretion

- Формат комментариев для сломанных URL
- Как именно сравнивать старый тег и main для Anton111111
