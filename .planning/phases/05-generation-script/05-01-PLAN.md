  <action>
Создать Python-скрипт `generate.py` в корне репозитория со следующей структурой и функциональностью:

**Константы (вверху файла):**
- `BASE_CONFIG = Path(__file__).parent / 'config.yaml'`
- `OUTPUT_DIR = Path(__file__).parent / 'output'`
- `PERSONAL_OUTPUT = OUTPUT_DIR / 'config-personal.yaml'`
- `WORK_OUTPUT = OUTPUT_DIR / 'config-work.yaml'`
- `ADULT_START = '# >>> ADULT'`
- `ADULT_END = '# <<< ADULT'`
- `ADULT_KEYWORDS = ['pornhub', 'stripchat', 'chaturbate', 'bongacams', 'onlyfans', 'fansly', 'hentai', 'rule34', 'nsfw', 'porn']`

**ANSI-цвета:**
- Включить ANSI на Windows: `if os.name == 'nt': os.system('')`
- Константы: GREEN, RED, YELLOW, BOLD, RESET

**Функции:**

1. `parse_args() -> argparse.Namespace` -- argparse с `--dry-run` флагом. Описание на русском.

2. `read_config(path: Path) -> list[str]` -- Читает файл с encoding='utf-8-sig' (автоматически убирает BOM). Возвращает список строк с сохранением переносов. Если файл не найден -- ошибка и exit(1).

3. `filter_adult_blocks(lines: list[str]) -> tuple[list[str], list[dict]]` -- State machine: `inside_adult` флаг. При встрече `# >>> ADULT` (stripped) -- включить пропуск, запомнить номер строки начала. При встрече `# <<< ADULT` -- выключить пропуск, записать блок в список удалённых `{start, end, lines_count}`. Строки маркеров тоже удаляются (не попадают в результат). После цикла: если `inside_adult` всё ещё True -- записать предупреждение о непарном маркере. Возвращает (отфильтрованные строки, список удалённых блоков).

4. `validate_yaml(content: str) -> tuple[bool, str | None]` -- `yaml.safe_load(content)`. Если исключение -- вернуть (False, str(exception)). Если ok -- (True, None). Import yaml с try/except ImportError для сообщения "pip install pyyaml".

5. `check_adult_keywords(lines: list[str]) -> dict[str, list[int]]` -- Для каждого keyword из ADULT_KEYWORDS проверить каждую строку (case-insensitive). Вернуть `{keyword: [номера_строк_где_найдено]}`. Пустой список = keyword не найден.

6. `backup_if_exists(path: Path) -> bool` -- Если файл существует, копировать в `path.with_suffix(path.suffix + '.bak')` через `shutil.copy2()`. Вернуть True если бэкап создан.

7. `write_output(path: Path, lines: list[str], dry_run: bool) -> None` -- Создать `path.parent` (os.makedirs exist_ok=True). Если не dry_run: backup_if_exists, затем записать файл (encoding='utf-8'). Если dry_run: только вывести что файл был бы создан.

8. `print_report(...)` -- Печатает детальный отчёт на русском с ANSI-цветами:
   - Заголовок: "=== Генерация конфигов mihomo ==="
   - Источник: путь к config.yaml, кол-во строк
   - Раздел "Удалённые adult-блоки": для каждого блока -- строки start-end, кол-во строк
   - Итого удалено: N строк в M блоках
   - Раздел "Проверка adult-ключевых слов в work-конфиге": для каждого keyword -- GREEN "OK" или RED "НАЙДЕНО" с номерами строк
   - Раздел "Валидация YAML": для каждого файла -- GREEN "OK" или RED "ОШИБКА" + описание
   - Раздел "Результат": файлы созданы или dry-run
   - Если dry_run: пометить "[DRY RUN] Файлы не созданы"

9. `main()` -- Главная функция:
   - `args = parse_args()`
   - `lines = read_config(BASE_CONFIG)`
   - `work_lines, removed_blocks = filter_adult_blocks(lines)`
   - Проверить непарные маркеры (warnings)
   - `write_output(PERSONAL_OUTPUT, lines, args.dry_run)` -- personal = полная копия
   - `write_output(WORK_OUTPUT, work_lines, args.dry_run)` -- work = отфильтрованный
   - Валидация YAML обоих файлов (если не dry_run -- читать созданные файлы, если dry_run -- валидировать содержимое в памяти)
   - `keyword_results = check_adult_keywords(work_lines)`
   - `print_report(...)` -- вывести весь отчёт
   - Определить exit code: errors список -- непарные маркеры, найденные adult-keywords, невалидный YAML. `sys.exit(1 if errors else 0)`

**Важные детали:**
- Язык вывода: русский
- Кодировка чтения: utf-8-sig (автоудаление BOM)
- Кодировка записи: utf-8
- Все пути через pathlib.Path
- Скрипт работает на Python 3.x (совместимость 3.8+, но целевая -- 3.14)
- Shebang: `#!/usr/bin/env python3`
- Docstring модуля: на русском, объясняет назначение
